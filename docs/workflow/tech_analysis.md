# Анализ технологий для информационной системы «Sigma Tech»

## 1. Состав приложений

Информационная система состоит из трёх приложений:
- **Сервер (бэкенд)** – реализует бизнес-логику, API, работу с БД.
- **Веб-клиент для покупателей** – интернет-магазин: каталог, корзина, оформление заказа, личный кабинет.
- **Веб-клиент для администратора/менеджеров** – панель управления: товары, заказы, клиенты, отчёты.

Все приложения используют единую базу данных PostgreSQL. Обмен данными – REST API поверх HTTPS, формат JSON.

## 2. Анализ технологий для сервера (бэкенд)

**Параметры сравнения:**
1. Скорость разработки (наличие готовых решений, ORM, документация).
2. Производительность (асинхронность, потребление ресурсов).
3. Поддержка типизации и валидации данных (снижает число ошибок).

| Технология           | Параметр 1 | Параметр 2 | Параметр 3 | Итоговая оценка (1–5) |
|----------------------|------------|------------|------------|----------------------|
| Python + FastAPI     | 5          | 5          | 5          | 15/15                |
| Python + Django      | 5          | 3          | 4          | 12/15                |
| Node.js + Express    | 3          | 4          | 3          | 10/15                |

**Выбор: Python + FastAPI**  
**Обоснование:**  
- Команда владеет Python.  
- FastAPI асинхронный, имеет встроенную валидацию через Pydantic и автоматическую генерацию OpenAPI-документации.  
- Позволяет быстро создать API для всех прецедентов.  

## 3. Анализ технологий для веб-клиентов (оба клиента)

Оба клиента – веб-приложения, работают в браузере. Для них выбран одинаковый стек для унификации и скорости разработки.  
**Параметры сравнения:**
1. Скорость разработки (компонентный подход, сборка, горячая перезагрузка).
2. Производительность в браузере.
3. Экосистема (библиотеки для UI, роутинга, работы с API).

| Технология           | Параметр 1 | Параметр 2 | Параметр 3 | Итоговая оценка |
|----------------------|------------|------------|------------|------------------|
| React + JS/SCSS      | 4          | 4          | 5          | 13/15            |
| Vue + JS/SCSS        | 5          | 4          | 4          | 13/15            |
| Angular + TypeScript | 3          | 5          | 4          | 12/15            |

**Выбор: React + JavaScript + SCSS**  
**Обоснование:**  
- React является самым популярным, много готовых компонентов (таблицы, формы, корзина).  
- Легко создавать отдельные приложения для покупателя и администратора, переиспользуя общие компоненты.  
- Команда имеет базовые знания JavaScript. SCSS позволяет удобно писать стили.  

## 4. Анализ СУБД

**Параметры сравнения:**
1. Поддержка сложных запросов (JOIN, агрегации, подзапросы для финансовых отчётов).
2. Надёжность и соответствие ACID.
3. Простота настройки и администрирования.

| СУБД         | Параметр 1 | Параметр 2 | Параметр 3 | Итоговая оценка |
|--------------|------------|------------|------------|------------------|
| PostgreSQL   | 5          | 5          | 4          | 14/15            |
| MySQL        | 4          | 4          | 5          | 13/15            |
| SQLite       | 2          | 3          | 5          | 10/15            |

**Выбор: PostgreSQL**  
**Обоснование:**  
- Лучшая поддержка сложных запросов и аналитических функций (необходимо для расчёта скидок и отчётов).  
- Полная ACID-совместимость, надёжность.  
- Бесплатная, широко используется в индустрии.  
- Хорошая интеграция с FastAPI через asyncpg или SQLAlchemy.

## 5. Протоколы и форматы обмена

- **Протокол:** HTTP/HTTPS.  
- **Стиль API:** REST.  
- **Формат данных:** JSON.  
- **Обоснование:**  
  - REST является стандартом для веб-сервисов.  
  - JSON легко читается и генерируется на любом языке (Python, JavaScript).  
  - Можно использовать стандартные HTTP-методы (GET, POST, PUT, DELETE).
